## 2.  Compare Spring and Springboot? What are the benfits of Srpingboot?
- Spring Framework: Spring is a comprehensive framework for building enterprise-grade Java applications. It provides a wide range of features, including dependency injection, aspect-oriented programming, transaction management, and various other modules like Spring MVC, Spring Security, Spring Data, etc. Spring allows for extensive customization and flexibility but requires significant configuration and setup.
- Spring Boot : Spring Boot is an extension of the Spring framework designed to simplify the development of new Spring applications. It does so by offering: 
- 1. Auto-Configuration: Automatically configures Spring applications based on the dependencies you include in your project. 
- 2. Standalone Applications: Allows creating standalone applications with embedded servers like Tomcat, Jetty, or Undertow, eliminating the need for a separate application server.
- 3. Production-Ready Features: Provides built-in features like metrics, health checks, and externalized configuration. 
- 4. Convention Over Configuration: Favors sensible defaults to reduce the need for boilerplate code and configuration.



## 3.  What is IOC and What is DI?
- Inversion of Control (IoC) is a design principle used in software engineering where the control of objects or portions of a program is transferred to a container or framework. Itâ€™s a technique to achieve loose coupling in software design.
- Dependency Injection (DI) is a specific implementation of the IoC principle. It involves passing (or injecting) dependencies (objects or services that a class needs to function) into a class instead of the class creating them itself.

## 4.  What is  @CompnonentScan ?
- The @ComponentScan annotation is used in Spring to specify the base packages to scan for annotated components, which include @Component, @Service, @Repository, @Controller, and other stereotype annotations. It tells Spring where to look for these components to be automatically detected and registered as Spring beans.

## 5.  What is  @SpringbootApplication ?
- The @SpringBootApplication annotation is a convenience annotation in Spring Boot that combines several other annotations and configurations commonly used in Spring applications. It marks the main class of a Spring Boot application and enables several important features.

## 7.  What is default bean name for  @Component  and  @Bean ? Also compare  @Component  and  @Bean
- Default Bean Name for @Component : When a class is annotated with @Component (or its specializations like @Service, @Repository, @Controller), the default bean name is the class name with the first letter in lowercase.
- Default Bean Name for @Bean : When a method is annotated with @Bean, the default bean name is the name of the method.

## 9.  Explain  @Autowired ,  @Qualifier  ,  @Resource  and  @Primary ?

- @Autowired: Used for automatic dependency injection by type.
- @Qualifier: Used with @Autowired to specify which bean to inject when multiple beans of the same type exist.
- @Resource: Used for dependency injection by name.
- @Primary: Used to give precedence to a bean when multiple beans of the same type are present, resolving ambiguity.

## 10.  How many annotaitons we can use to inject a bean?

- @Autowired:

Used for automatic dependency injection by type.
Can be applied to fields, constructors, or setter methods.

- @Qualifier:

Used along with @Autowired to specify which bean to inject when multiple beans of the same type exist.
Specifies the bean name or qualifier value.

- @Resource:

Used for dependency injection by name.
Can be applied to fields, setter methods, or constructors.
Resolves the bean by the name specified in the name attribute of the annotation.

- @Inject:

Similar to @Autowired, part of the Java Dependency Injection (JSR-330) standard.
Provides more fine-grained control over dependency injection compared to @Autowired.
Requires javax.inject dependency.

- @Value:

Used to inject values from properties files or environment variables.
Can inject simple values like strings, numbers, or booleans into beans.

## 15.  Configure a bean using xml. If bean has parameters/dependencies, how can we configure the bean?

- Create the Interface and Classes
```java
// UserService.java
public class UserService {
    private UserRepository userRepository;

    // Constructor with dependency injection
    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    // Method using userRepository
    public void saveUser(User user) {
        userRepository.save(user);
    }
}

```


- Create XML Configuration File
```xml
 <!-- Define the bean for UserRepository -->
    <bean id="userRepository" class="com.example.UserRepositoryImpl"/>

    <!-- Define the bean for UserService with constructor injection -->
    <bean id="userService" class="com.example.UserService">
        <constructor-arg ref="userRepository"/> <!-- Inject userRepository bean -->
    </bean>

```


- Using the Configured Beans
```java
import org.springframework.context.ApplicationContext;
import org.springframework.context.support.ClassPathXmlApplicationContext;

public class MainApp {
    public static void main(String[] args) {
        ApplicationContext context = new ClassPathXmlApplicationContext("applicationContext.xml");

    
        UserService userService = context.getBean("userService", UserService.class);

    
        userService.saveUser(new User("John Doe"));

        
        ((ClassPathXmlApplicationContext) context).close();
    }
}
```
